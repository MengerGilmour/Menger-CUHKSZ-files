羊驼毛：强大的、可复制的指令遵循模型

作者： Rohan Taori*、 Ishaan Gulrajani*、 张天一*、 Yann Dubois*、 Xuechen Li*、 Carlos Guestrin、 Percy Liang、 Tatsunori B. Hashimoto

我们介绍Alpaca 7B，这是一个在 52K 指令跟踪演示上对 LLaMA 7B 模型进行微调的模型。在我们对单轮指令跟踪的初步评估中，Alpaca 的行为在质量上与 OpenAI 的 text-davinci-003 相似，同时出人意料地小且易于/廉价复制（<600 美元）。查看我们在GitHub上发布的代码。
更新：公共演示现已禁用。发布演示的最初目标是以一种易于理解的方式传播我们的研究成果。我们觉得我们已经基本实现了这个目标，考虑到托管成本和内容过滤器的不足，我们决定取消演示。

概述

GPT-3.5 (text-davinci-003)、ChatGPT、Claude 和 Bing Chat 等指令跟踪模型已经变得越来越强大。现在，许多用户定期与这些模型进行交互，甚至将它们用于工作。然而，尽管广泛部署，指令遵循模型仍然存在许多缺陷：它们可能生成虚假信息、传播社会刻板印象并产生有毒语言。

为了在解决这些紧迫问题上取得最大进展，学术界的参与非常重要。不幸的是，在学术界对指令跟踪模型进行研究一直很困难，因为没有易于访问的模型在功能上接近闭源模型（例如 OpenAI 的 text-davinci-003）。

我们正在发布有关指令跟踪语言模型（称为Alpaca）的发现，该模型是根据 Meta 的LLaMA 7B 模型进行微调的。我们使用 text-davinci-003以自指令的方式生成 52K 指令跟踪演示来训练 Alpaca 模型。在自指导评估集上，Alpaca 显示了许多与 OpenAI 的 text-davinci-003 类似的行为，但也令人惊讶地小且易于/廉价地复制。

我们正在发布我们的训练配方和数据，并打算在未来发布模型权重。我们还举办了一个互动演示，以使研究界能够更好地了解羊驼的行为。交互可以暴露意想不到的功能和故障，这将指导我们对这些模型的未来评估。我们还鼓励用户在我们的网络演示中报告任何相关行为，以便我们更好地理解和减轻这些行为。由于任何版本都存在风险，我们将在本博文的后面部分讨论我们对于此公开版本的思考过程。

我们强调Alpaca仅用于学术研究，禁止任何商业用途。这个决定有三个因素：首先，Alpaca 是基于 LLaMA 的，它具有非商业许可，所以我们必然继承这个决定。其次，指令数据基于OpenAI的text-davinci-003，其使用条款禁止开发与OpenAI竞争的模型。最后，我们还没有设计足够的安全措施，因此羊驼还没有准备好投入一般用途。

训练食谱

在学术预算下训练高质量的指令跟踪模型面临两个重要挑战：强大的预训练语言模型和高质量的指令跟踪数据。最近发布的 Meta 的新 LLaMA 模型解决了第一个挑战。对于第二个挑战，自指导论文建议使用现有的强语言模型来自动生成指令数据。特别是，Alpaca 是一种语言模型，使用 LLaMA 7B 模型的监督学习对 OpenAI 的 text-davinci-003 生成的 52K 指令跟踪演示进行微调。

下图说明了我们是如何得到羊驼模型的。对于数据，我们通过基于自指令方法生成了遵循指令的演示。我们从自指令种子集中的175 个人工编写的指令输出对开始。然后，我们提示 text-davinci-003 使用种子集作为上下文示例生成更多指令。我们通过简化生成管道（参见GitHub中的详细信息）改进了自指令方法，并显着降低了成本。我们的数据生成过程产生 52K 个独特的指令和相应的输出，使用 OpenAI API 的成本不到 500 美元。

配备了这个遵循指令的数据集，然后我们使用 Hugging Face 的训练框架微调 LLaMA 模型，并利用完全分片数据并行和混合精度训练等技术。在我们的初始运行中，在 8 个 80GB A100 上微调 7B LLaMA 模型需要 3 个小时，这对于大多数云计算提供商而言成本不到 100 美元。我们注意到，可以通过提高培训效率来进一步降低成本。

初步评估

为了评估 Alpaca，我们对自指导评估集的输入进行了人工评估（由 5 名学生作者进行）。该评估集由自学指导作者收集，涵盖各种面向用户的指导，包括电子邮件写作、社交媒体和生产力工具。我们对 text-davinci-003 和 Alpaca 7B 进行了盲法成对比较，发现这两个模型具有非常相似的性能：Alpaca 在与 text-davinci-003 的比较中以 90 比 89 获胜。

考虑到较小的模型尺寸和适度的指令跟随数据量，我们对这个结果感到非常惊讶。除了利用这个静态评估集之外，我们还以交互方式测试了 Alpaca 模型，发现 Alpaca 在多种输入上的表现通常与 text-davinci-003 类似。我们承认我们的评估可能在规模和多样性方面受到限制。因此，我们发布了 Alpaca 的互动演示，并鼓励读者对 Alpaca 进行自我评价并向我们提供反馈。

已知的限制

羊驼还表现出语言模型的一些常见缺陷，包括幻觉、毒性和刻板印象。幻觉似乎是羊驼的一种常见失败模式，甚至与 text-davinci-003 相比也是如此。

发布决定

我们相信，发布上述资产将使学术界能够对遵循指令的语言模型进行受控科学研究，从而产生更好的科学成果，并最终找到解决这些模型现有缺陷的新技术。

同时，任何发布都会带来一定的风险。首先，我们认识到发布我们的培训方案揭示了某些功能的可行性。一方面，这使得更多的人（包括不良行为者）能够创建可能造成伤害（无论是有意还是无意）的模型。另一方面，这种意识可能会激励迅速采取防御行动，尤其是学术界，现在有能力对此类模型进行更深入的安全研究。总的来说，我们认为这一特定版本对研究界的好处大于风险。

鉴于我们正在发布训练配方，考虑到配方的简单性，我们相信发布数据、模型权重和训练代码所带来的进一步风险最小。同时，发布这些资产对于可重复科学具有巨大的好处，以便学术界可以使用标准数据集、模型和代码来执行受控比较并探索扩展。

为 Alpaca 部署交互式演示也会带来潜在风险，例如更广泛地传播有害内容以及降低垃圾邮件、欺诈或虚假信息的门槛。我们制定了两项风险缓解策略。首先，我们使用OpenAI 的内容审核 API实现了内容过滤器，该过滤器可以过滤掉 OpenAI 使用策略定义的有害内容。其次，我们使用 Kirchenbauer 等人中描述的方法对所有模型输出进行水印 。2023，以便其他人可以（以一定概率）检测输出是否来自 Alpaca 7B。最后，我们对使用演示版有严格的条款和条件；它仅限于非商业用途以及遵循LLaMA 许可协议的用途。

我们知道，一旦我们发布模型权重或者用户训练自己的指令跟踪模型，这些缓解措施就可以被规避。然而，通过安装这些缓解措施，我们希望推进最佳实践，并最终为基础模型的负责任部署制定社区规范。

未来发展方向

我们对羊驼释放的研究机会感到兴奋。未来有许多令人兴奋的方向：

  • 评估：我们需要更严格地评估羊驼。我们将从HELM （语言模型的整体评估）开始，希望它能够发展以捕获更多生成性、遵循指令的场景。
  • 安全性：我们希望进一步研究羊驼的风险，并通过自动红队、审计、自适应测试等方法提高其安全性。
  • 理解：我们希望更好地理解能力是如何从培训方案中产生的。您需要基本模型的哪些属性？当你扩大规模时会发生什么？需要指令数据的哪些属性？在 text-davinci-003 上使用自我指令的替代方法是什么？
